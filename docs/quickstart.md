# Quickstart — Conversion OS

**Goal: a populated, governed business vault with a first deliverable in ~10 minutes, on zero infra.**

## What you need
- Claude (Claude Code or Cowork).
- An empty folder you control — ideally inside a synced drive (Google Drive / Dropbox) so it's backed up and shareable.
- Git (optional but recommended — it's your version history and safety net).

## Steps
1. Point Claude at the empty folder.
2. Run **`/setup`** (Conversion OS Setup).
3. Pick a profile: **Solo**, **Team**, or **Agency**.
4. Paste your website URL and drop a few docs (deck, pricing, about). Setup reads them first so it only *confirms* what it can't infer.
5. Answer a handful of skippable questions (one-liner, offers, ICP, voice, top goal).
6. Setup scaffolds the vault, seeds your context + first KPI baselines, and hands you a **first deliverable** plus a **report card** (what's solid / thin / missing).

## Your daily loop
- Morning: **`/daily`** → today's brief and Top 3.
- As things happen: **`/meeting-capture`**, **`/pipeline-update`**, drop anything into `Inbox/`.
- Friday: **`/weekly-review`**. Monthly: **`/business-review`** (proves the trend).
- Occasionally: **`/optimizer`** to keep the vault clean.

## Where things live
Open `CLAUDE.md` at the vault root — it's the map. Every kind of information has one canonical home, listed in its routing table.
