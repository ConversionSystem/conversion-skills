# Daily Use

The OS is only as good as the rhythm you run it on. Here is the rhythm.

## The loop

- **Each morning:** `/daily`. A brief and an argued Top 3.
- **As things happen:** `/meeting-capture` after a call, `/pipeline-update` after a deal moves, or drop anything into `Inbox/`.
- **Friday:** `/weekly-review`. Wins, what dragged, next week.
- **Monthly:** `/business-review`. The whole ledger rolled against a baseline. This is the proof the OS is working.
- **Now and then:** `/optimizer`. Keeps the vault fast, clean, and honest.

You do not have to run all of it. Start with `/daily` and `/meeting-capture`. Add the rest as it sticks.

## A solo week

- Monday morning: `/daily`. Plan the week against your top goal.
- After each sales call: `/meeting-capture`. Decisions and actions get filed, metrics hit the ledger.
- Wednesday: `/lead-research` on three target accounts, then `/email-personalize` to draft the outreach. You send it.
- Thursday: `/newsletter` to draft this week's issue. You publish it.
- Friday: `/weekly-review`. End of month: `/business-review`.

## An agency client cycle

Every client lives in its own firewalled workspace. Work one client at a time.

1. `/client new` to onboard a client. A short interview, then a clean workspace.
2. `/client brief` before a call. One screen of context.
3. `/meeting-capture` (or `/client log`) after the call.
4. `/seo-audit` or `/ads-audit` to find the work. `/seo-optimize`, `/landing-page`, `/blog-post` to draft it. You ship it.
5. `/client report` weekly or monthly. Numbers first, from that client's ledger.
6. `/client health` to catch risk early. `/churn-watch` across the book. `/portfolio-watch` for the owner's roll-up.

The firewall is enforced. No skill reads another client. `scripts/check-firewall.sh` proves it.

## A team

- Set seats with `/team`. Each person gets a profile, an access file, and a task list.
- `/business-review` rolls up the whole company.
- Permissions are real: AI scoping plus storage access plus an audit trail. See `confidentiality.md`.

## Hands-off: the operator

`/operator` builds a daily run that reads the OS, pulls from the tools you connect, writes the day's synthesis, and escalates the few things that need a human. It drafts. It never sends. You can schedule it. See `connectors.md`.

## How it compounds

Every run that moves a number adds one row to `Memory/kpi-ledger.md`. Nothing is erased. Run the same skill next month and the ledger shows the delta. `/business-review` turns that history into a trend. That trend is the reason to keep the OS, and the reason it is worth paying for.
